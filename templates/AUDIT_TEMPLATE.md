## Obligation-level compliance audit

This section comes before prose diagnostics. Preserve the level assigned by authority; do not infer severity from distance from planning.

| Control / instruction / boundary | Authority source | Planned level | Draft execution | Required meaning preserved? | Free execution used safely? | Result |
|---|---|---|---|---|---|---|
|  |  | 1–5 |  | yes / no | yes / no / not applicable | pass / blocker / evidence-based craft review / neutral difference |

Required checks:

- [ ] Every Level 1 hard fact, ownership decision, mechanics limit, legal state and consent boundary passes.
- [ ] Every Level 2 semantic, knowledge and causal result is delivered, even when wording differs.
- [ ] Every Level 3 protective boundary blocks the named invalid implication or result.
- [ ] Level 4 targets are evaluated through actual local evidence, not treated as automatic blockers.
- [ ] Level 5 wording, minor staging, optional reactions and natural synonyms remain free unless they create demonstrated damage.
- [ ] Repetition across work artifacts did not raise authority.
- [ ] No audit diagnosis silently created a new story rule.
- [ ] Version comparison distinguishes lost facts from equivalent wording and neutral execution changes.

Severity rules:

- **blocker:** actual Level 1–3 failure affecting authority, ownership, knowledge, mechanics, relationship state, legal state, consent, causal result or approval safety;
- **major:** sustained structural or voice damage without hard-fact failure;
- **important:** local evidence, rhythm, clarity or character-specificity issue worth revision;
- **optional:** preference or alternative execution with no demonstrated damage.

A Level 4 or Level 5 divergence cannot be a blocker solely because it differs from planning.

### Required/result-bearing blocker cases

- only a dossier path, trait, accent or voice label is supplied;
- a required application has no productive consequence or restrictive boundary;
- required ownership fails removal or replacement without reclassification or justification;
- approved-event residue or existing ties disappear so the character can react only to Mielle;
- practical competence required for the result is ignored;
- later-canon knowledge, maturity or relationship state leaks backward;
- a substantial revision lacks required comparison and second-order reread.

### Ensemble and optional-texture blocker cases

- every present character receives a compulsory beat for coverage;
- optional texture steals ownership or introduces unsupported knowledge;
- individually optional beats accumulate into a demonstrated cast-roll-call, repeated-function or flattened-dialogue cluster;
- meaningful witnesses are forced to speak merely to prove presence;
- the audit treats a removable optional line as automatically superficial.

The following do not fail by themselves:

- a meaningful witness remaining silent;
- a background character having no line;
- a removable optional texture beat;
- unused practical competence when no natural or result-bearing trigger exists;
- equivalent wording or different minor staging.

A passing Character Voice receipt does not automatically pass Character Dossier Application.

## Show-don't-tell evidence audit

Unsupported telling is a blocking issue when it asserts a material emotional, relational, tactical, inferential, power, injury, danger, authority, knowledge or consequence claim without evidence.

| Claim in prose | Obligation / material type | On-page evidence | Pass / blocker | Repair |
|---|---|---|---|---|
|  | Level 1–3 / emotion / relation / combat / inference / power / authority / knowledge / consequence |  | pass / blocker |  |

Blocking cases:

- emotional label without behavior when the label carries a material claim;
- relationship label without changed behavior or approved status;
- Holmes conclusion without clue and live uncertainty;
- combat no-hit without cause;
- power effect without mechanism;
- exposition without front-story trigger;
- thematic or moral ending that repeats a conclusion already delivered.

Do not block a direct private thought merely because it names an emotion when the chapter has earned and authorized it.

### Receipt validity

The receipt is invalid when:

- a pre-draft risk is omitted;
- a universal cluster has only `pass`, `not present`, or equivalent without local evidence or an inspected pattern;
- a substantial revision lacks version comparison;
- corrected paragraphs were not reread;
- the report inventories surface words without identifying root cause;

- specificity regression confuses a neutral Level 4–5 change with loss of a Level 1–3 fact;
- a Level 4–5 preference is promoted into a blocker;
- sanitation gate is incomplete;
- analysis is generic enough to fit an unrelated chapter unchanged.
