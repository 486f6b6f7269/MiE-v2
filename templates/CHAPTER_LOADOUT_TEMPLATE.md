Complete before the decision-to-authority check and before prose.

| Instruction / result / boundary | Authority source | Level in plan | Level in mini delta | Level in scene cards / pre-check | Required meaning | Free execution | Status |
|---|---|---|---|---|---|---|---|
|  |  | 1–5 | same / justified return | same / justified return |  |  | pass / return to planning |

Required checks:

- [ ] Every Level 1–3 control names an authority source.
- [ ] No Level 4 target became a mandatory outcome, exact line, required reaction or lexical prohibition.
- [ ] Level 5 wording and minor staging remain free.
- [ ] Repetition across artifacts did not raise authority.
- [ ] Any changed level returned to the owning authority and was not invented here.
- [ ] Character presence is classified as required/result-bearing, meaningful witness, optional texture or background.
- [ ] Removal and replacement tests are interpreted by presence class and obligation level.
- [ ] Preferred nouns, gestures, joke counts or sentence patterns are not treated as hard rules without authority.
- [ ] Pre-draft guard separates Level 1–3 blockers from Level 4 craft pressure and Level 5 freedom.

A generic `closed vs free checked` statement is invalid.

## Decision-to-authority check

Include every Level 1–3 chapter decision, material fact or visible reaction that could conflict with another source. Add Level 4–5 rows only when a local craft choice risks implying a harder fact.

| Planned decision or fact | Obligation level | Source / planning authority | Governing character, power, relationship or setting rule | Continuity and witness limit | What the draft may show | What the draft must not imply | Status |
|---|---|---|---|---|---|---|---|
|  | 1–5 |  |  |  |  |  | pass / return to planning |

Rules:

- Cite exact files and useful headings or entries.
- A row with an unresolved Level 1–3 conflict blocks drafting.
- `No conflict noticed` is invalid.
- Do not create a row for every decorative detail.
- In non-combat chapters, include result-bearing payments, invitations, refusals, accepted help, object transfers, information withheld and relationship non-results.
