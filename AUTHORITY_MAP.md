# Authority map

## File domains

- `canon/` contains stable rules and durable facts.
- `continuity/` contains the current state, known information, active emotional residue, open threads and event memory.
- `work/` contains temporary material only. Nothing there is canon unless promoted after chapter approval.
- `chapters/approved/` contains final story text. Only approved chapter text may promote new facts into continuity.
- `templates/` contains formats only, never active canon.
- `archive/` contains superseded or historical material that no longer controls production.

## Conflict order

If two files conflict, apply this order:

1. Newest explicit user decision.
2. Approved chapter text.
3. Current continuity.
4. Stable canon.
5. Work material.
6. Templates and archive material.

## Source boundary

The original Critical Role transcript determines original canon events, decisions, knowledge flow and consequences. It is not a prose script. `canon/SOURCE_ADAPTATION_RULES.md` defines adaptation method.

## Promotion boundary

A plan does not become canon. A draft does not update memory. An audit does not rewrite story facts. Only approved chapter text may promote facts into continuity.
